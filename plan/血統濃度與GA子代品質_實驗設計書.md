# 基因演算法中「血統濃度」與子代品質相關性研究 — 實驗設計書

範圍定位：專題 ～ 碩士論文之間

---

## 1. 研究背景與文獻定位

### 1.1 既有研究脈絡

雖然沒有研究直接使用「血統濃度」一詞，但本題目站在三條既有研究線的交會點上：

1. **親子適應度相關性 / Evolvability**
   Manderick, de Weger & Spiessens (1991) 最早提出統計量來衡量交配運算子在親代與子代間保留適應度資訊的程度；後續研究（如 TSP 上不同 crossover 運算子的比較）發現不同運算子在「親代→子代」適應度相關性上表現差異很大。Altenberg 進一步將此能力定義為「evolvability」：運算子/編碼方式產生比親代更優秀子代的能力。
   → 這條線只看「直接雙親」，未追溯多代祖先。

2. **適應度地景的欺騙性（Deception）與 Building Block 理論**
   在高度 epistatic（基因交互作用強）或具欺騙性（deceptive）的地景上，適應度地景可以平滑卻難搜尋，也可以崎嶇卻容易搜尋——崎嶇程度本身不能決定 GA 的搜尋難度，關鍵在基因間交互作用。這解釋了「兩個適應度都不高的親代，仍可能透過交叉重組各自貢獻互補的『好積木』，產生比雙親都優秀的子代」的理論基礎（Holland 的 Building Block Hypothesis；Trap function / NK landscape 文獻）。
   → 這條線解釋了「壞 × 壞 = 好」為何可能發生，但通常只討論單代。

3. **血緣/祖先貢獻追蹤（Genealogy / Ancestry Contribution）**
   - 在 GP（基因規劃）的家系分析研究中發現：初始群體中只有極少數個體在最終代仍有存活後代，且只有一小部分祖先真正貢獻基因到最佳解（McPhee, Burlacu, Donatucci 等人的研究）。
   - 動物育種學（quantitative genetics）早有成熟公式：Boichard et al. (1997) 提出計算「每個祖先對目前世代的邊際貢獻」，所有祖先貢獻加總為 1，用以找出「有效祖先數（effective number of ancestors）」，是保種與血統純度評估的標準工具。
   - 演化式 AI 領域近期（RHEA framework, Meyerson et al. 2024）也用遞迴方式定義「個體的祖先集合」與「最終解中有多少基因比例來自某起始個體」，但目的是歸功給人類專家，**並未檢驗「祖先品質的血統濃度」與「最終解品質」之間的相關性**。

### 1.2 本研究的定位與貢獻

> 把（2）動物育種學的「祖先貢獻百分比」公式，正式引入（1）親子適應度相關性研究的框架中，並系統性地在不同「育種策略」（血統濃度高低）下，檢驗其與 GA 收斂表現的因果關係。

這是目前文獻中尚未被系統檢驗的缺口，適合作為專題/論文題目。

## 2. 研究問題與假設

**研究問題（RQ）：** 在基因演算法中，個體的「血統濃度」（祖先品質的加權濃度）是否能預測該個體及其子代的解品質？不同的育種策略（精英 vs. 混合 vs. 劣質 vs. 隨機）對最終收斂表現有何影響，且此效應是否隨問題的「欺騙性/epistasis」程度而改變？

**假設：**
- **H1**：血統濃度（LC）與個體適應度之間存在正相關，且此相關性在標準 GA（無特殊育種策略）中即可觀察到。
- **H2**：純精英育種（只用最優親代）短期內收斂速度最快，但因群體多樣性快速流失，長期更容易陷入早熟收斂，最終解品質不一定優於混合育種策略。
- **H3**：即使在「純劣育種」（只用最差親代交配）的極端條件下，仍會出現少數子代優於其雙親（因路徑表示法中，劣質個體仍可能share 部分優良子路徑），但其出現機率與幅度顯著低於混合或精英育種。

## 3. 核心指標：血統濃度（Lineage Concentration, LC）

### 3.1 血統佔比（Ancestry Proportion, AP）— 遞迴定義

設初始族群（第 0 代）的每個個體為「始祖（founder）」，編號 f = 1...N。
對任一個體 i，定義其血統佔比向量 AP_i = {p_i,1, p_i,2, ..., p_i,N}，其中 p_i,f 表示個體 i 的基因有多少比例可追溯回始祖 f，且 Σ_f p_i,f = 1（即全部血統佔比加總為 100%）。

- **始祖本身**：AP_f = {f: 1.0}（自己 100% 是自己的血統）
- **第 g 代個體 i，父母為 a、b（透過交叉產生）**：
  ```
  p_i,f = 0.5 × p_a,f + 0.5 × p_b,f   對所有 f
  ```
  （此處假設交叉運算子平均地混合雙親基因，與動物育種學「血統百分比」算法的標準假設一致；若使用的 crossover 機制明顯不對稱，可在實作時依比例調整權重，並在論文中說明此為簡化假設與其限制。）

### 3.2 始祖品質分數（Founder Quality, q_f）

對每個始祖 f，依其在初始族群中的適應度排名正規化為 [0,1] 區間：
```
q_f = 1 − (rank(f) − 1) / (N − 1)      # rank 1 = 最佳個體 → q = 1
```

### 3.3 血統濃度（Lineage Concentration, LC）

個體 i 的血統濃度定義為「其所有祖先品質的血統加權平均」：
```
LC_i = Σ_f  p_i,f × q_f
```
LC_i ∈ [0, 1]，數值越高代表該個體的血統越「集中於優良始祖」。

### 3.4 範例（4 個始祖，簡化說明）

| 始祖 | 適應度排名 | q_f |
|---|---|---|
| f1 | 1（最佳） | 1.00 |
| f2 | 2 | 0.67 |
| f3 | 3 | 0.33 |
| f4 | 4（最差） | 0.00 |

若第 3 代個體 X 的血統佔比為 AP_X = {f1: 0.5, f2: 0.25, f3: 0.25, f4: 0}，
則 LC_X = 0.5×1.00 + 0.25×0.67 + 0.25×0.33 + 0×0.00 = **0.75**（血統濃度偏高）。

### 3.5 補充指標（選用，增加分析深度）

- **血統多樣性（Shannon entropy of AP_i）**：衡量個體血統是「集中於少數始祖」還是「廣泛混雜」，可用來區分「高 LC 但血統單一」與「高 LC 但血統多元」兩種情況對表現的不同影響。
- **有效始祖數（effective number of founders）**：1 / Σ_f (p_i,f)²，直接借用動物育種學中評估群體血統純度/瓶頸效應的標準指標。

## 4. 實驗設計

### 4.1 測試問題：旅行推銷員問題（TSP）

選擇 TSP 作為主測試問題，原因：(1) 是 GA 文獻中最常用、最具代表性的基準問題；(2) 有現成公開的標準測試集 **TSPLIB**，方便與既有文獻比較；(3) 路徑式編碼天然適合觀察「子路徑（building block）」如何在交叉中重組，貼合 H3 的理論動機。

建議起手用中小型實例，方便大量重複實驗（30+ 次/組）：
- `eil51`（51 個城市，已知最優解）
- `kroA100`（100 個城市）
- 若計算資源允許，可加入 `pr1002` 等大型實例做延伸驗證

### 4.2 GA 基本設置

| 項目 | 設定 |
|---|---|
| 編碼方式 | 路徑表示法（permutation encoding） |
| 交叉運算子 | Order Crossover（OX）或 Partially Mapped Crossover（PMX） |
| 突變運算子 | Swap mutation（交換兩個城市），機率 0.01～0.05 |
| 族群大小 | 100～200（須記錄足夠的祖先資訊，不宜過大） |
| 世代數 | 300～500（依問題規模調整，以收斂為準） |
| 菁英保留 | 每代保留 top 1～2 個體直接進入下一代（避免最佳解流失） |
| 重複次數 | 每組設定跑 30 組不同隨機種子（確保統計檢定力） |

### 4.3 自變項：四種育種策略

| 組別 | 親代選擇方式 | 預期血統濃度趨勢 |
|---|---|---|
| **A. 精英育種** | 每代僅從最優 20% 個體中選取雙親 | 持續走高，可能快速同質化 |
| **B. 標準 GA（基準組）** | 一般競賽選擇（tournament selection, size=3） | 中等、自然變動 |
| **C. 劣質育種** | 每代僅從最差 20% 個體中選取雙親（極端壓力測試） | 持續走低 |
| **D. 隨機交配** | 雙親完全隨機選取，不考慮適應度 | 接近族群平均，作為「無選擇壓力」對照 |

> 方法論注意：A、C 組因選擇壓力極端，組內 LC 變異會很小（大家血統都差不多高/低），不利於做「LC vs. 適應度」的相關性分析。因此：
> - **H1 的相關性檢定**建議用**所有組別合併**的資料做混合效應回歸（mixed-effects regression），以 LC 為自變項、子代適應度為依變項、組別為控制變項，這樣才能取得足夠的 LC 變異範圍。
> - **H2、H3 的策略效果比較**則直接用 A/B/C/D 四組的最終解品質做 ANOVA 比較。

### 4.4 依變項 / 量測指標

1. 每代最佳/平均路徑長度（tour length）
2. 族群多樣性（平均成對 Hamming 距離 或 edge-difference 多樣性）
3. 每個個體的 LC 值（依 3.3 節公式即時計算）
4. **LC 與子代適應度的 Pearson / Spearman 相關係數**（核心檢定指標，對應 H1）
5. 最終代最佳解品質（與已知最優解的差距百分比）
6. 收斂世代數（達到最優解 X% 以內所需代數）
7. （延伸）「逆勢子代」出現率：子代適應度優於其雙親平均值的比例，依組別比較（對應 H3）

### 4.5 統計分析方法

- 相關分析：Pearson/Spearman correlation（LC vs. 個體適應度），跨組別、跨世代彙整
- 群組比較：one-way ANOVA + post-hoc（如 Tukey HSD）比較 A/B/C/D 最終解品質
- 若要更嚴謹：以混合效應模型（mixed-effects model）控制「隨機種子」與「世代」為隨機效應，避免重複量測造成的統計謬誤
- 顯著水準建議 α = 0.05，並報告效果量（effect size，如 Cohen's d 或 η²）而非僅看 p 值

## 5. 可能的延伸實驗（若時間允許，提升論文深度）

在 TSP 之外，額外加入一個「人工可控 epistasis 強度」的測試問題（例如 **NK landscape**，可調參數 K 控制基因交互作用強度，或 **Trap function**），用以驗證：

> H1 的「血統濃度 vs. 子代品質」正相關性，是否會隨著問題的 epistasis/欺騙性程度增加而減弱？

這能呼應第 1.1 節提到的 deception 理論，讓研究不只是「TSP 上的單一觀察」，而是能討論「血統濃度效應的適用邊界」，更有理論貢獻，是碩論等級很合適的第二章延伸。

## 6. 實作注意事項（效能與工程細節）

1. **祖先字典會隨代數增長**：每個個體的 AP 字典理論上可包含所有始祖，但實務上多數始祖的貢獻會隨代數迅速趨近於 0。建議設定門檻（例如 p_i,f < 0.001 即捨去並重新正規化），避免記憶體爆炸，同時不影響 LC 計算精度。
2. **計算時機**：LC 可以在每次產生子代時，順手用雙親的 AP 字典算出（O(始祖數) 複雜度），不需要每次都重新追溯整個家系樹。
3. **資料記錄格式建議**：每個個體記錄 `(id, parent_a_id, parent_b_id, AP_dict, fitness, generation)`，方便事後重建任意子代的完整家系與 LC 趨勢圖。
4. **視覺化建議**：用桑基圖（Sankey diagram）或堆疊面積圖呈現「始祖血統佔比隨世代的消長」，是論文中很有說服力的圖。
