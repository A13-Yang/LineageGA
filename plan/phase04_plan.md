# Phase 4：分析與視覺化 — 統計檢定 + 圖表 + Web UI 儀表板

**時程：第 7-8 週**
**前置依賴：Phase 3（完整實驗數據已產出）**

---

## 目標

完成所有統計分析（驗證 H1/H2/H3）、產出 8 種研究圖表（中文標註），並建立互動式 Web UI 儀表板供即時查看與探索實驗數據。

---

## 任務清單

### Week 7：統計分析 + 靜態圖表

- [ ] **T4.1** 實作 `analysis/statistics.py` — 統計檢定模組
  - **H1 檢定：LC 與適應度相關性**
    - Pearson / Spearman 相關係數（全資料合併）
    - 依世代分層計算趨勢（每 50 代一個切片）
    - 回傳 r 值、p 值、95% 信賴區間
  - **H2 檢定：四策略最終解品質比較**
    - One-way ANOVA
    - Tukey HSD post-hoc 兩兩比較
    - 效果量：η²（eta-squared）
  - **H3 檢定：逆勢子代分析**
    - 各策略逆勢子代出現率的描述統計
    - Kruskal-Wallis 非參數檢定（若分布不正態）
  - 所有檢定結果存為結構化 `StatisticalReport` dataclass

- [ ] **T4.2** 實作 `analysis/visualize.py` — 靜態圖表（中文標註、PNG 300 DPI）
  - **圖 1：收斂曲線**
    - 四策略並列，每代最佳適應度
    - 30 runs 的 mean ± std 帶狀圖（`fill_between`）
    - 標註已知最優解水平線
  - **圖 2：LC 分布演化圖**
    - 每 50 代一個箱型圖，四策略用不同顏色
    - 或用小提琴圖呈現分布形狀
  - **圖 3：LC vs. Fitness 散點圖**
    - 散點 + 線性回歸線
    - 標註 r 值與 p 值
    - 依策略分面（facet）
  - **圖 4：族群多樣性曲線**
    - 四策略的每代 edge-difference 多樣性
    - 觀察精英組多樣性崩潰速度

- [ ] **T4.3** 實作進階圖表
  - **圖 5：桑基圖**（Plotly 互動式）
    - 選取單次 run，呈現始祖血統佔比隨世代的消長
    - X 軸 = 世代（每 50 代取樣），Y 軸 = 始祖，色帶寬度 = AP 佔比
    - 精英組 vs. 隨機組各一張
  - **圖 6：逆勢子代率曲線**
    - 四策略每代的逆勢子代出現率
    - 30 runs 平均 ± std
  - **圖 7：有效始祖數曲線**
    - 每代平均有效始祖數
    - 觀察血統瓶頸效應
  - **圖 8：熱力圖**
    - X = 始祖品質（q_f 分組），Y = 世代
    - 色彩 = 該品質群組的平均 AP 佔比

- [ ] **T4.4** 圖表風格統一設定
  - 中文字體設定（`matplotlib.rcParams` 設定 SimHei / Microsoft JhengHei）
  - 統一配色方案（四策略固定四色）
  - 圖表大小、字體大小、線寬統一

### Week 8：Web UI 互動式儀表板

- [ ] **T4.5** 選定 Web 框架並建立骨架
  - 使用 **Plotly Dash**（因已用 Plotly 畫桑基圖，生態系統一致）
  - 建立 `dashboard/app.py` 主程式
  - 基本佈局：側邊欄（篩選器）+ 主區域（圖表）

- [ ] **T4.6** 儀表板功能頁面
  - **頁面 1：實驗總覽**
    - 四策略的收斂曲線（互動式，可 hover 看數值）
    - 最終解品質柱狀圖 + 誤差棒
    - ANOVA 結果摘要卡片
  - **頁面 2：血統分析**
    - LC 分布隨世代演化的動畫 slider
    - LC vs. Fitness 互動式散點圖（可選世代範圍）
    - 桑基圖（可選 run / 策略）
  - **頁面 3：詳細數據**
    - 可篩選的數據表格（DataTable）
    - 下載原始數據功能（CSV）

- [ ] **T4.7** 儀表板互動功能
  - 下拉選單：選擇 TSP 實例（eil51 / kroA100）
  - 多選框：選擇要比較的策略
  - Slider：選擇世代範圍
  - 即時更新圖表（Dash callback）

- [ ] **T4.8** 實作 `analysis/report.py` — 自動報告產生
  - 讀取實驗數據 → 跑統計檢定 → 產出所有圖表 → 存至 `data/results/figures/`
  - 一鍵執行：`python -m analysis.report`

## 風險與注意事項

| 風險 | 對策 |
|---|---|
| matplotlib 中文顯示亂碼 | 事先測試字體設定，使用系統已安裝的中文字體 |
| 桑基圖節點過多（200 始祖）難以閱讀 | 只顯示 AP > 5% 的始祖，其餘歸入「其他」 |
| Dash 啟動後佔用端口衝突 | 預設 port=8050，可設定為可配置參數 |
| 大量數據的互動式圖表卡頓 | 數據預聚合後再送至前端，避免傳輸原始個體級數據 |
| ANOVA 前提（常態性、變異數同質性）不滿足 | 同時跑 Kruskal-Wallis 非參數版本作為備選 |
